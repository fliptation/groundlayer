# Contributing to Ground Layer

Ground Layer belongs to everyone who contributes. There's no gatekeeping, no application process. If you want to help, you're in.

## Ways to Contribute

### Improve the Content
- Fix typos, unclear language, or outdated information
- Add real-world examples and case studies
- Suggest new sections or topics
- Submit a pull request or open an issue

### Share Local Examples
- Document cooperatives, land trusts, or community projects in your area
- Write up how a local food network, energy cooperative, or governance experiment was started
- Share lessons learned — what worked, what didn't

### Translate
- Help make Ground Layer accessible in every language
- Adapt content for your cultural context
- Review existing translations

### Build Tools
- Improve the website (design, accessibility, performance)
- Build interactive tools (mapping, visualization, planning)
- Create templates for local implementation
- Develop educational materials

### Implement Locally
- Take these ideas and put them into practice where you live
- Document your process and share what you learn
- Connect with other communities doing similar work

### Spread the Word
- Share with friends, family, and communities
- Present at local meetups or events
- Write about it
- Connect it to existing movements

## Guidelines

- **Be respectful.** We welcome all perspectives. Disagree with ideas, not people.
- **Be practical.** Prioritize real-world examples over theory. If something works, share how.
- **Be clear.** Write for a global audience. Avoid jargon. Explain acronyms. Use simple language.
- **Be constructive.** If you see a problem, suggest a solution.
- **Be inclusive.** These ideas need to work for everyone — not just the privileged few.

## How to Submit Changes

1. Fork the repository
2. Create a branch for your changes
3. Make your edits
4. Submit a pull request with a clear description of what you changed and why

For larger changes, open an issue first to discuss your idea.

## Content Structure

- `docs/` — The system design documents (markdown)
- `website/` — The public website (Next.js)
- `CONTRIBUTING.md` — This file

## Code of Conduct

Be kind. Be constructive. Assume good intent. We're all here because we want a better world. Let's act like it.
